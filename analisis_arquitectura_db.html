Seguridad y RBAC

Tabla Propósito Relación / Dependencia
mae_usuario Credenciales y estado de acceso. PK: id_usuario
mae_rol Perfiles funcionales (Admin, Mues, etc). PK: id_rol
mae_permiso Acciones granulares (ej: FI_CREAR). PK: id_permiso

Lógica de Negocio

Al loguearse, el backend recupera los roles de rel_usuario_rol y cruza con rel_rol_permiso. Esto genera un token JWT con los permisos "empaquetados" que la UI usa para ocultar/mostrar elementos.

GESTIÓN

Módulo URS (Solicitudes)

Tabla Uso Principal Campo Clave
mae_solicitud Ticket central de trámite. datos_json (Esquema variable)
mae_solicitud_tipo Configuración de flujos y UI. formulario_config
mae_solicitud_comentario Hilo de chat y eventos. id_solicitud (FK)

Lógica de Flexibilidad

El sistema URS es agnóstico del tipo de solicitud. Los campos de un "Traspaso de Equipo" viven dentro del JSON de la solicitud, permitiendo crear nuevos tipos de trámites desde la base de datos sin tocar el código fuente.

Flujo de Sincronización

Las fichas se crean y agendan en App_Ma_Agenda_MUESTREOS. La APP MAM descarga estas agendas, recolecta datos en los detalles, y al sincronizar, inserta los valores finales en App_Ma_Resultados marcando la ficha como 'ENVIADA'.
