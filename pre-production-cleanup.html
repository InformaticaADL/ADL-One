🧹 Limpieza pre-producción — ADL ONE

Guía de truncado de datos transaccionales antes del go-live
Base de datos: ADL_ONE_DB
⚠️ Antes de ejecutar: hacé backup completo de ADL_ONE_DB. Este script borra datos transaccionales en forma irreversible. El script está particionado en transacciones por bloque para que, si algo falla, solo ese bloque haga rollback.

Archivo SQL:

db-migrations/pre-production-cleanup.sql

📊 Resumen general

Se borran (transaccionales)
Se conservan (maestros)
Decisión del negocio

Se borra todo lo operativo creado durante el QA: fichas, agenda, equipos asignados a muestreos, resultados, rutas, ejecuciones, solicitudes URS, chats, notificaciones, logs de login y tokens de reset de password, y opcionalmente el log de auditoría. Se conservan los maestros: usuarios, roles, permisos, catálogos de negocio (empresas, centros, normativas, técnicas, laboratorios, muestreadores, equipos…) y la configuración del sistema (menú, eventos de notificación, tipos de solicitud).

🟠 Decisiones que debes tomar Antes de ejecutar

# Decisión Detalle Comportamiento default
1 ¿Conservar logs de auditoría? El bloque 7 borra App_Audit_Log. Si querés mantener el registro de quién hizo qué durante QA (útil para demo o post-mortem), comentá el DELETE en ese bloque. Borrar
2 ¿Conservar grupos de rutas creados en QA? Por defecto NO se borra mae_grupos_rutas. Si creaste grupos solo para probar y no querés que aparezcan en prod, descomentá la línea correspondiente en el bloque 2. Conservar
3 ¿Equipos de prueba en el catálogo? NO se borra mae_equipo ni mae_equipo_catalogo. Si en QA se crearon equipos solo para probar mezclados con equipos reales, debes borrar manualmente uno por uno. Conservar
4 ¿Muestreadores de prueba? NO se borra mae_muestreador. Si hay muestreadores ficticios, eliminarlos manualmente. Conservar
5 ¿Fichas legítimas creadas durante QA? El script borra TODAS las fichas. Si hay alguna ficha real de cliente creada durante QA, exportala antes o filtrá el WHERE en el bloque 1. Borrar todo
6 ¿Historial de versiones de equipos? NO se borra mae_equipo_historial por defecto. Si en QA hubo cambios de versión sobre equipos de prueba, descomentá esa línea en el bloque 7. Conservar

🚀 Pasos para ejecutar

  1. Backup completo de ADL_ONE_DB. No negociable.
  2. Apagar el backend (api-backend-adlone) para que no inserte filas mientras corre el script.
  3. Abrir SQL Server Management Studio o Azure Data Studio.
  4. Revisar las decisiones de la sección anterior y descomentar / comentar las líneas opcionales.
  5. Ejecutar db-migrations/pre-production-cleanup.sql. Cada bloque corre en su propia transacción.
  6. Revisar los contadores que imprime al final — los transaccionales deben quedar todos en 0.
  7. Reiniciar el backend.
  8. Todos los usuarios deben volver a hacer login (sus JWTs vigentes quedaron inválidos porque permisos_version volvió a 0).
  9. Hacer una prueba humo (login, crear ficha, ver detalle) para confirmar que la app responde.
  10. ✅ Listo para producción.
